    1. Validating forms:
      Form validation can be observed when a user enters data on a web page, or form.
      When entering any incorrect data, the web throws an alert message, indicating messages
      such as the following:
      "This field is required"
      "Please enter a valid email"
      "the data entered must be only numbers", etc.
      Data validation, both on the client and server-side, helps security on websites. When the user
      enters the information correctly, the application allows the data to be sent to the server
      by storing it, usually in a database. Otherwise, if the information entered is incorrect,
      the application will not allow the data to be sent to the server.
      Form validation mainly serves three things:
      1-Obtain the correct data in the correct way, so that in this way it is stored in the correct format.
      2- Protect user data, to protect your information.
      3- Protect ourselves.
      Form validation can be done in HTML, in a default way, or Java Script, in a more personalized way.
      The data can have a pseudo-class of: valid or: invalid Example
    2. Using Fetch: Basically, it is an interface for the use of Ajax
      (Asynchronous JavaScript and XML) with Javascript. An API is used to access
      server resources asynchronously, using on 'promises'. The promise object,
      returned from fetch () does not it will be rejected with an error status
      even if it is an HTTP 404 or 500 error. Instead, it will be resolved with
      an 'ok' status set to 'false'.
      How to perform a fetch () request:
      fetch ('http://example.com/movies.json')
      .then (response => response.json ())
      .then (data => console.log (data));
      A fetch () request will be rejected with 'TypeError' when a network error is encountered.
